# Folder: backend/app

> Trigger: When this folder's structure/responsibilities/file list changes, update this document.
> 触发条件:当本文件夹的结构/职责/文件列表变化时，更新此文档。

Application code owns domain models, API controllers, admin resources/pages, providers, services, and Excel DTOs. / 应用代码负责领域模型、API 控制器、后台资源/页面、服务、提供者与 Excel DTO。
Controllers stay thin; services own validation, transactions, cache, progressive stage rules, imports, and exports. / 控制器保持轻薄；服务负责校验、事务、缓存、递进阶段规则、导入与导出。
Business rules must remain reusable by API, admin, tests, exports, and queues. / 业务规则必须可被 API、后台、测试、导出和队列复用。

## Files
- `Enums/`: backend/app/Enums - typed lifecycle values / 类型化生命周期值
- `Exports/`: backend/app/Exports - Excel export classes / Excel 导出类
- `Filament/`: backend/app/Filament - admin panel resources and singleton settings pages / 后台面板资源与单例设置页面
- `Http/`: backend/app/Http - API controllers and requests / API 控制器与请求
- `Imports/`: backend/app/Imports - Excel import adapters / Excel 导入适配器
- `Models/`: backend/app/Models - Eloquent domain models / Eloquent 领域模型
- `Providers/`: backend/app/Providers - Laravel and Filament providers / Laravel 与 Filament 提供者
- `Services/`: backend/app/Services - registration, progressive stage, cache, and import business services / 报名、递进阶段、缓存与导入业务服务
